证明自动驾驶比人安全，我们还需要500年吗？根据兰德智库，自动驾驶系统至少要积累177亿公里的里程测试，这需要100辆车24小时不断测试500年才能实现。而自动仿真系统的诞生，终于让车企在实现日行百万公里测试的同时，模拟各类极端情况，为自动驾驶的安全问题提供了更多解决思路。

目前主流的自动驾驶仿真系统多是根据游戏引擎开发，这也让游戏领域的巨头在自动仿真驾驶领域占有不少天然优势，比如今年由腾讯游戏学堂举办的2021腾讯游戏开发者大会上就已经出现了与自动驾驶相关的专题演讲；随着BAT和华为的入局，仿真系统会成为自动驾驶领域的最后一块技术高地吗？

本期节目，腾讯自动驾驶仿真技术总监孙驰天，与「声动活泼」联合创始人徐涛共同探讨，游戏引擎如何应用在自动驾驶领域，自动驾驶仿真领域有哪些重要进展，这个领域又将在老玩家和新玩家的参与下走向何方。

▼延伸阅读
UnrealOpenDay2020 虚幻引擎助力自动驾驶模拟仿真 | 腾讯自动驾驶 孙驰天
AADS: Augmented autonomous driving simulation using data-driven algorithms
ISO/PAS 21448:2019 Road vehicles — Safety of the intended functionality
工业仿真：通过计算机图形、虚拟现实等技术方式，与云计算、AI结合，将实体工业的各个环节通过数字形式，在虚拟环境中实现重建和交互，目前已在航天军工、汽车交运等行业普及。
游戏引擎：指游戏开发时可复用、可拓展的软件组件，这个概念最早出现 20 世纪 90 年代的 id Software 推出的 FPS 游戏《毁灭战士》。游戏开发商在获得游戏引擎的授权后，无需对引擎做出大量修改，就可以通过较低成本创造相似的新游戏。游戏引擎目前已经大量应用在仿真测试系统的开发。
Unknown Unsafe：指ISO21448「预期功能安全」标准下「未知的非安全场景」，由于自动驾驶应用的场景具有的随机性和复杂性，设计自动驾驶功能时，存在需求难定义、评价难量化的问题，是安全风险的主要来源之一。该场景下，驾驶员如果不及时介入，容易发生交通事故。
